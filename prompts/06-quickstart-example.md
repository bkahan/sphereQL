# Prompt: Create a Polished "5-Minute Quickstart" Example for Python

## Context

SphereQL's Python package (after prompts 01, 02, 04 are done) exposes:
- `Pipeline(categories, embeddings)` for in-memory semantic analysis.
- `VectorStoreBridge` for plugging into existing vector DBs.
- `visualize()` for 3D interactive visualization.

But there's no single, copy-pasteable example that a developer can run
in 5 minutes to understand what SphereQL does and why they'd use it.

The goal is a standalone Python script that:
1. Requires ZERO external services (no Qdrant, no Pinecone, no OpenAI).
2. Uses a small built-in dataset with pre-computed embeddings.
3. Demonstrates every major feature in sequence.
4. Prints clear, annotated output explaining what's happening.
5. Generates the 3D visualization as the finale.

## Deliverables

### 1. Create `sphereql-python/examples/quickstart.py`

```python
"""
SphereQL Quickstart — run this script to see SphereQL in action.

No external services or API keys required. Uses a built-in dataset
of 100 sentences across 10 topics with pre-computed embeddings.

Usage:
    pip install sphereql
    python quickstart.py
"""
```

The script should follow this narrative arc:

**Act 1: "What does my knowledge look like?" (Visualization)**
- Load the built-in dataset (see section 2 below).
- Call `sphereql.visualize()` to generate the 3D sphere.
- Print a message explaining what the user is seeing.

**Act 2: "Find me something similar" (Nearest Neighbor Search)**
- Pick a query sentence from the dataset.
- Run `pipeline.nearest(query_embedding, k=5)`.
- Print the query and results with distances.
- Highlight that the results are from the correct topic.

**Act 3: "What are the natural groupings?" (Glob Detection)**
- Run `pipeline.detect_globs(k=None, max_k=10)`.
- Print each glob with its top categories and member count.
- Point out how globs correspond to topics.

**Act 4: "How do I get from A to B?" (Concept Path)**
- Pick two items from different topics (e.g., a science sentence and
  a cooking sentence).
- Run `pipeline.concept_path(source_id, target_id, graph_k=10)`.
- Print the path step by step, showing how concepts bridge topics.

**Act 5: "What's the local shape here?" (Local Manifold)**
- Run `pipeline.local_manifold(query_embedding, neighborhood_k=15)`.
- Print the variance ratio and explain what it means.

**Act 6: "Plug into my existing vector DB" (Bridge Demo)**
- Create an `InMemoryStore`, upsert the same data.
- Build a `VectorStoreBridge`.
- Run `hybrid_search` and show results.
- Run `sync_projections` and show the count.
- Print a message: "In production, replace InMemoryStore with
  QdrantStore or PineconeStore to add SphereQL to your existing stack."

Each act should be separated by a clear header and 1-2 sentences of
context explaining WHY this feature is useful, not just what it does.

### 2. Create built-in dataset at `sphereql-python/examples/dataset.py`

This file contains:
- `SENTENCES`: list of 100 dicts with `{text, category, embedding}`.
  - 10 categories, 10 sentences each.
  - Categories: science, technology, cooking, sports, music, history,
    nature, health, philosophy, business.
  - Embeddings: 64-dimensional vectors generated by a deterministic
    hash-based encoder (same approach as the Rust e2e test in
    `sphereql-vectordb/tests/e2e_text_to_sphereql.rs`).

The encoder function:
```python
def encode(text: str, dim: int = 64) -> list[float]:
    """Deterministic bag-of-words encoder via FNV-1a hashing."""
    STOPWORDS = {"the", "and", "with", "for", "its", "this", "that", ...}
    bag = [0.0] * dim
    for token in text.lower().split():
        token = ''.join(c for c in token if c.isalpha())
        if len(token) < 3 or token in STOPWORDS:
            continue
        h = 0xcbf29ce484222325
        for b in token.encode():
            h ^= b
            h = (h * 0x100000001b3) & 0xFFFFFFFFFFFFFFFF
        bag[h % dim] += 1.0
    mag = sum(x*x for x in bag) ** 0.5
    if mag > 0:
        bag = [x / mag for x in bag]
    return bag
```

Sentences should be real, interesting, and clearly distinguishable by
topic. Not lorem ipsum. Not "this is a sentence about science." Write
sentences that sound like they came from Wikipedia or a textbook.

Example:
- science: "The double-slit experiment demonstrates that light exhibits
  both wave and particle properties."
- cooking: "Caramelizing onions over low heat for forty minutes develops
  deep, sweet flavors that transform a simple soup."
- history: "The Treaty of Westphalia in 1648 established the principle
  of state sovereignty in international relations."

## Constraints

- ZERO external dependencies beyond `sphereql` itself. No numpy, no
  sentence-transformers, no requests. The encode function uses only
  stdlib.
- The script must run in under 5 seconds on a modern laptop.
- All output goes to stdout. The only file created is the HTML
  visualization.
- The script must work with Python 3.8+.
- Write clear, idiomatic Python. Use f-strings. No type: ignore comments.
- The dataset sentences must be factually accurate and interesting —
  they're the first thing a potential user will read.
